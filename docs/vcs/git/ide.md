## Общие сведения

Работа с Git через интегрированные инструменты в IDE упрощает процесс разработки, предоставляя визуальный интерфейс для основных операций с репозиторием.

## 1С: EDT

1С: EDT (Eclipse-based Development Tools) — официальная среда разработки для 1С:Предприятие 8, основанная на Eclipse и имеющая встроенную поддержку Git через плагин EGit.

### Настройка Git в 1С: EDT

1. Откройте настройки: Window → Preferences (или Eclipse → Preferences на macOS)
2. Перейдите в раздел Team → Git
3. Укажите путь к исполняемому файлу Git (если не определяется автоматически)
4. Настройте имя пользователя и email для коммитов в разделе Team → Git → Configuration

### Основные операции в 1С: EDT

#### Инициализация репозитория

1. Откройте проект в 1С: EDT
2. В контекстном меню проекта выберите Team → Share Project
3. Выберите Git и нажмите Next
4. Отметьте "Use or create repository in parent folder of project" или укажите путь
5. Нажмите Finish

#### Клонирование репозитория

1. В меню выберите File → Import
2. Выберите Git → Projects from Git
3. Выберите Clone URI
4. Введите URL репозитория
5. Выберите ветки для клонирования
6. Укажите папку назначения
7. Выберите способ импорта проекта

#### Просмотр изменений

1. Откройте Git Staging view: Window → Show View → Other → Git → Git Staging
2. В панели "Unstaged Changes" отображаются все измененные файлы
3. Двойной клик на файле откроет diff в редакторе

#### Добавление файлов в индекс (Stage)

1. В Git Staging view найдите измененный файл в "Unstaged Changes"
2. Перетащите файл в "Staged Changes" или используйте кнопку "+"
3. Или используйте контекстное меню файла → Team → Add to Index

#### Создание коммита

1. Добавьте файлы в индекс (Stage)
2. Введите сообщение коммита в поле "Commit Message"
3. Нажмите кнопку "Commit" или используйте Ctrl+Enter
4. Убедитесь, что сообщение соответствует [соглашению о коммитах](../storage.md#соглашение-о-коммитах)

#### Отправка изменений (Push)

1. В Git Staging view нажмите кнопку "Push" (стрелка вверх)
2. Или используйте контекстное меню проекта → Team → Remote → Push
3. Если ветка еще не отслеживается удаленно, 1С: EDT предложит настроить upstream

#### Получение изменений (Pull)

1. В Git Staging view нажмите кнопку "Pull" (стрелка вниз)
2. Или используйте контекстное меню проекта → Team → Pull

#### Переключение веток

1. В Git Repositories view (Window → Show View → Git Repositories) найдите репозиторий
2. Раскройте Branches → Local
3. Выберите ветку и используйте контекстное меню → Checkout
4. Или используйте контекстное меню проекта → Team → Switch To → выберите ветку

#### Разрешение конфликтов

1. При возникновении конфликтов 1С: EDT отобразит их в Git Staging view
2. Откройте файл с конфликтом — он будет помечен специальными маркерами
3. Используйте встроенные инструменты для разрешения:
   - Team → Merge Tool — открыть инструмент слияния
   - В редакторе: Accept Current Change / Accept Incoming Change / Accept Both
4. После разрешения всех конфликтов добавьте файл в индекс и создайте коммит

### Интеграция с GitHub/GitLab

1С: EDT поддерживает интеграцию с популярными хостингами Git:

1. В Git Repositories view используйте контекстное меню репозитория → Configure Push/Pull
2. Добавьте remote (origin) с URL репозитория
3. Для работы с Pull Requests можно использовать расширения или веб-интерфейс

## VS Code

VS Code также широко используется для разработки на 1С и имеет отличную поддержку Git.

### Настройка Git в VS Code

1. Откройте настройки (Ctrl+,)
2. Найдите "Git: Path" и укажите путь к Git (если необходимо)
3. Настройте "Git: User Name" и "Git: User Email"

### Основные операции в VS Code

Операции аналогичны 1С: EDT, но интерфейс может немного отличаться:

- **Панель Source Control**: Ctrl+Shift+G
- **Просмотр истории**: В панели Source Control → "..." → "View History"
- **Создание ветки**: В нижней панели статус-бара → клик на название ветки → "Create new branch"
- **Слияние веток**: "..." → "Branch" → "Merge Branch..."

## Рекомендации для 1С программистов

### Работа с конфигурацией 1С

При работе с конфигурацией 1С через Git рекомендуется:

1. **Использовать выгрузку в файлы** — конфигурация должна быть выгружена в формат XML или EDT
2. **Настроить .gitignore** — исключить временные файлы и служебные папки (см. [.gitignore](gitignore.md))
3. **Атомарные коммиты** — каждый коммит должен содержать логически связанные изменения
4. **Осмысленные сообщения** — описание изменений должно быть понятным для команды

### Работа с расширениями

При разработке расширений конфигурации:

1. Каждое расширение — отдельный репозиторий или подмодуль
2. Используйте ветвление для изоляции изменений
3. Перед коммитом проверяйте код с помощью встроенных инструментов 1С: EDT

### Синхронизация с хранилищем 1С

Если проект использует и Git, и хранилище 1С:

1. Синхронизируйте изменения регулярно
2. Используйте инструменты типа Oproxy для автоматической синхронизации
3. Следите за тем, чтобы версии в Git и хранилище 1С соответствовали друг другу

## Полезные расширения

### Для 1С: EDT

- Встроенная поддержка Git через EGit (входит в состав)
- Расширения для работы с GitHub/GitLab через Marketplace

### Для VS Code

- **GitLens** — расширенная функциональность Git
- **Git Graph** — визуализация истории коммитов
- **Git History** — просмотр истории файлов

## Частые проблемы и решения

### Git не найден

**Проблема**: IDE не может найти исполняемый файл Git

**Решение**:

1. Установите Git с официального сайта
2. Укажите путь к `git.exe` в настройках IDE
3. Перезапустите IDE

### Конфликты при слиянии

**Проблема**: Возникли конфликты при попытке слияния веток

**Решение**:

1. Используйте встроенные инструменты разрешения конфликтов
2. При сложных конфликтах используйте внешние инструменты (например, Beyond Compare)
3. Всегда тестируйте код после разрешения конфликтов

### Неправильный формат коммита

**Проблема**: Коммит не соответствует соглашению о коммитах

**Решение**:

1. Используйте функцию "Amend" для изменения последнего коммита
2. Или создайте новый коммит с правильным форматом
3. Настройте шаблоны коммитов в IDE
