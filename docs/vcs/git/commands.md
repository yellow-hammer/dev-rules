## Общие сведения

Этот раздел содержит справочник основных команд Git, наиболее часто используемых при разработке на 1С.

## Настройка

### Первоначальная настройка

```bash
# Установка имени пользователя
git config --global user.name "Иванов Иван"

# Установка email
git config --global user.email "ivanov@example.com"

# Установка редактора по умолчанию
git config --global core.editor "code --wait"  # VS Code
git config --global core.editor "notepad"      # Блокнот

# Установка имени ветки по умолчанию
git config --global init.defaultBranch main

# Просмотр всех настроек
git config --list

# Просмотр конкретной настройки
git config user.name
```

## Работа с репозиторием

### Инициализация и клонирование

```bash
# Инициализация нового репозитория
git init

# Инициализация с указанием ветки
git init -b main

# Клонирование репозитория
git clone <url>

# Клонирование в указанную папку
git clone <url> <папка>

# Клонирование конкретной ветки
git clone -b <ветка> <url>

# Клонирование с ограниченной глубиной (shallow clone)
git clone --depth 1 <url>
```

### Просмотр состояния

```bash
# Статус репозитория
git status

# Краткий статус
git status -s

# Статус с игнорируемыми файлами
git status --ignored
```

## Работа с файлами

### Добавление файлов

```bash
# Добавить все изменения
git add .

# Добавить конкретный файл
git add <файл>

# Добавить все файлы определенного типа
git add *.bsl

# Интерактивное добавление
git add -i

# Добавить все изменения включая удаленные файлы
git add -A

# Добавить только измененные файлы (не новые)
git add -u
```

### Удаление файлов

```bash
# Удалить файл из Git и файловой системы
git rm <файл>

# Удалить файл только из Git (оставить в файловой системе)
git rm --cached <файл>

# Удалить папку рекурсивно
git rm -r <папка>
```

### Переименование и перемещение

```bash
# Переименовать файл
git mv <старое_имя> <новое_имя>

# Переместить файл
git mv <файл> <новая_папка>/
```

## Коммиты

### Создание коммитов

```bash
# Создать коммит
git commit -m "сообщение"

# Создать коммит с телом
git commit -m "заголовок" -m "тело сообщения"

# Добавить все изменения и создать коммит
git commit -am "сообщение"

# Создать коммит с указанием автора
git commit -m "сообщение" --author="Имя <email>"

# Изменить последний коммит
git commit --amend -m "новое сообщение"

# Добавить изменения в последний коммит
git commit --amend --no-edit
```

### Просмотр коммитов

```bash
# История коммитов
git log

# Краткая история
git log --oneline

# История с графиком веток
git log --graph --oneline --all

# История конкретного файла
git log <файл>

# История с изменениями
git log -p

# История с статистикой
git log --stat

# История за период
git log --since="2 weeks ago"
git log --until="2024-01-01"

# История по автору
git log --author="Иванов"

# Поиск в сообщениях коммитов
git log --grep="обработка"
```

## Ветки

### Создание и переключение

```bash
# Просмотр веток
git branch

# Просмотр всех веток (включая удаленные)
git branch -a

# Создать ветку
git branch <ветка>

# Переключиться на ветку
git checkout <ветка>

# Создать и переключиться
git checkout -b <ветка>

# Создать ветку от конкретного коммита
git checkout -b <ветка> <коммит>

# Переименовать текущую ветку
git branch -m <новое_имя>

# Переименовать другую ветку
git branch -m <старое_имя> <новое_имя>
```

### Удаление веток

```bash
# Удалить ветку
git branch -d <ветка>

# Принудительное удаление
git branch -D <ветка>

# Удалить удаленную ветку
git push origin --delete <ветка>
```

## Слияние и перебазирование

### Слияние

```bash
# Слить ветку в текущую
git merge <ветка>

# Слияние без fast-forward
git merge --no-ff <ветка>

# Слияние только если возможно fast-forward
git merge --ff-only <ветка>

# Отменить слияние
git merge --abort
```

### Перебазирование

```bash
# Перебазировать текущую ветку
git rebase <ветка>

# Интерактивное перебазирование
git rebase -i <коммит>

# Продолжить перебазирование после разрешения конфликтов
git rebase --continue

# Отменить перебазирование
git rebase --abort

# Пропустить коммит при перебазировании
git rebase --skip
```

## Удаленные репозитории

### Просмотр и настройка

```bash
# Просмотр удаленных репозиториев
git remote -v

# Добавить удаленный репозиторий
git remote add <имя> <url>

# Изменить URL удаленного репозитория
git remote set-url <имя> <новый_url>

# Удалить удаленный репозиторий
git remote remove <имя>

# Переименовать удаленный репозиторий
git remote rename <старое_имя> <новое_имя>
```

### Получение и отправка

```bash
# Получить изменения
git fetch

# Получить из конкретного удаленного репозитория
git fetch <имя>

# Получить и слить
git pull

# Получить и слить из конкретной ветки
git pull <имя> <ветка>

# Отправить изменения
git push

# Отправить в конкретную ветку
git push <имя> <ветка>

# Отправить с установкой upstream
git push -u <имя> <ветка>

# Отправить все ветки
git push --all <имя>

# Отправить теги
git push <имя> --tags
```

## Отмена изменений

### Отмена в рабочей директории

```bash
# Отменить изменения в файле
git checkout -- <файл>

# Отменить все изменения
git checkout -- .

# Восстановить файл из индекса
git restore <файл>

# Восстановить все файлы
git restore .
```

### Отмена в индексе

```bash
# Убрать файл из индекса
git reset HEAD <файл>

# Убрать все файлы из индекса
git reset HEAD

# Убрать файл из индекса (новая команда)
git restore --staged <файл>
```

### Откат коммитов

```bash
# Откатить последний коммит (сохранить изменения)
git reset --soft HEAD~1

# Откатить последний коммит (удалить изменения из индекса)
git reset --mixed HEAD~1
# или
git reset HEAD~1

# Откатить последний коммит (удалить все изменения)
git reset --hard HEAD~1

# Откатить на N коммитов назад
git reset --hard HEAD~N

# Откатить к конкретному коммиту
git reset --hard <коммит>
```

## Stash (временное сохранение)

```bash
# Сохранить изменения
git stash

# Сохранить с описанием
git stash save "описание"

# Сохранить включая неотслеживаемые файлы
git stash -u

# Просмотр списка stash
git stash list

# Просмотр содержимого stash
git stash show

# Просмотр diff stash
git stash show -p

# Применить последний stash
git stash apply

# Применить конкретный stash
git stash apply stash@{0}

# Применить и удалить
git stash pop

# Удалить stash
git stash drop stash@{0}

# Удалить все stash
git stash clear
```

## Теги

```bash
# Создать легковесный тег
git tag <имя>

# Создать аннотированный тег
git tag -a <имя> -m "описание"

# Создать тег на конкретном коммите
git tag -a <имя> <коммит>

# Просмотр тегов
git tag

# Просмотр тегов с фильтром
git tag -l "v1.*"

# Просмотр информации о теге
git show <тег>

# Удалить тег
git tag -d <тег>

# Удалить удаленный тег
git push origin --delete <тег>

# Отправить тег
git push origin <тег>

# Отправить все теги
git push origin --tags
```

## Поиск и фильтрация

```bash
# Поиск в истории по содержимому
git log -S "текст"

# Поиск по регулярному выражению
git log -G "регулярное выражение"

# Поиск коммитов, изменяющих функцию
git log -L :<функция>:<файл>

# Поиск файлов
git ls-files | grep "паттерн"

# Поиск в рабочей директории
git grep "текст"

# Поиск в конкретной ветке
git grep "текст" <ветка>
```

## Дифф (сравнение)

```bash
# Показать изменения
git diff

# Показать изменения в индексе
git diff --staged
# или
git diff --cached

# Показать изменения между коммитами
git diff <коммит1> <коммит2>

# Показать изменения в файле
git diff <файл>

# Показать статистику изменений
git diff --stat

# Показать только имена измененных файлов
git diff --name-only
```

## Полезные команды

### Очистка

```bash
# Показать файлы для удаления
git clean -n

# Удалить неотслеживаемые файлы
git clean -f

# Удалить неотслеживаемые файлы и папки
git clean -fd

# Удалить игнорируемые файлы
git clean -fX
```

### Информация

```bash
# Показать информацию о репозитории
git remote show origin

# Показать последний коммит
git log -1

# Показать авторов и количество коммитов
git shortlog -sn

# Показать статистику по файлам
git ls-files | xargs wc -l
```

### Алиасы (примеры)

```bash
# Создать алиас
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.visual '!gitk'
```

## Рекомендации для 1С программистов

### Типичный рабочий процесс

```bash
# 1. Получить последние изменения
git pull

# 2. Создать ветку для задачи
git checkout -b feature/new-processing

# 3. Внести изменения в код

# 4. Проверить статус
git status

# 5. Добавить изменения
git add .

# 6. Создать коммит
git commit -m "feat: Добавлена новая обработка"

# 7. Отправить изменения
git push -u origin feature/new-processing

# 8. После code review - слить в main
git checkout main
git pull
git merge feature/new-processing
git push
```

### Проверка перед коммитом

```bash
# Просмотреть что будет закоммичено
git diff --staged

# Проверить форматирование (если настроено)
# Использовать Phoenix BSL для проверки кода 1С
```
