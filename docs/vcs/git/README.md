## Общие сведения

Git — распределенная система управления версиями, которая используется для отслеживания изменений в исходном коде и координации работы команды разработчиков.

В контексте разработки на 1С, Git может использоваться как альтернатива или дополнение к хранилищу конфигурации 1С. Работа с Git может осуществляться через интегрированные инструменты в IDE или напрямую через командную строку.

## Структура документации

Документация по работе с Git разделена на два основных раздела:

1. **[Работа с Git в IDE](ide.md)** — использование Git через графические интерфейсы в средах разработки (1C: EDT, VS Code, и другие)
2. **[Работа с Git через командную строку](cli.md)** — использование Git напрямую через терминал

## Когда использовать Git

Git рекомендуется использовать в следующих случаях:

- Разработка расширений конфигурации
- Разработка внешних обработок и отчетов
- Управление версиями скриптов и утилит
- Совместная работа над проектами с использованием современных практик разработки
- Интеграция с CI/CD системами

## Соглашение о коммитах

При работе с Git необходимо соблюдать [соглашение о коммитах](../storage.md#соглашение-о-коммитах), основанное на [Conventional Commits](https://www.conventionalcommits.org/ru/v1.0.0/).
