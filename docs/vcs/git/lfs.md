## Общие сведения

Git LFS (Large File Storage) — расширение Git, предназначенное для работы с большими файлами. Вместо хранения больших файлов непосредственно в репозитории, Git LFS сохраняет их на удаленном сервере и хранит в репозитории только указатели на эти файлы.

## Зачем нужен Git LFS

При работе с проектами 1С могут возникать ситуации, когда необходимо хранить в репозитории:

- Большие файлы выгрузки конфигурации
- Файлы внешних обработок и отчетов с большим размером
- Медиафайлы (изображения, видео)
- Архивы и дампы баз данных
- Другие бинарные файлы большого размера

Обычный Git не предназначен для работы с такими файлами, что может привести к:

- Медленной работе репозитория
- Превышению лимитов размера репозитория
- Долгому клонированию и получению изменений

## Установка Git LFS

### Windows

1. Скачайте установщик с [официального сайта Git LFS](https://git-lfs.github.com/)
2. Запустите установщик и следуйте инструкциям
3. После установки проверьте:

```bash
git lfs version
```

### Альтернативный способ (через Git)

Если Git установлен через установщик Git for Windows, Git LFS может быть уже включен. Проверьте:

```bash
git lfs version
```

Если команда не найдена, установите Git LFS отдельно.

## Инициализация Git LFS

После установки необходимо инициализировать Git LFS в репозитории:

```bash
git lfs install
```

Эта команда настраивает Git hooks для автоматической работы с LFS файлами. Команду нужно выполнить один раз для каждого репозитория (или глобально с флагом `--global`).

## Использование Git LFS

### Отслеживание файлов по типу

Указать Git LFS отслеживать файлы определенного типа:

```bash
# Отслеживать все файлы .erf (выгрузки конфигурации)
git lfs track "*.erf"

# Отслеживать все файлы .dt (дампы)
git lfs track "*.dt"

# Отслеживать файлы больше определенного размера
git lfs track "*.zip"
```

### Отслеживание конкретных файлов

```bash
# Отслеживать конкретный файл
git lfs track "path/to/large-file.bin"

# Отслеживать все файлы в папке
git lfs track "exports/*"
```

### Просмотр отслеживаемых файлов

```bash
# Просмотр всех паттернов отслеживания
git lfs track

# Просмотр содержимого .gitattributes
cat .gitattributes
```

### Коммит файлов

После настройки отслеживания работа с файлами происходит как обычно:

```bash
# Добавить файлы
git add .gitattributes
git add large-file.erf

# Создать коммит
git commit -m "feat: Добавлена выгрузка конфигурации через LFS"

# Отправить изменения
git push
```

Git LFS автоматически заменит большие файлы указателями при коммите.

## Работа с существующими репозиториями

### Миграция существующих файлов

Если в репозитории уже есть большие файлы, их нужно мигрировать в LFS:

```bash
# Миграция всех файлов определенного типа из истории
git lfs migrate import --include="*.erf" --everything

# Миграция файлов из конкретной ветки
git lfs migrate import --include="*.erf" --branch=main

# Миграция файлов больше определенного размера
git lfs migrate import --above=10MB --everything
```

**Важно:** После миграции история коммитов будет переписана. Это требует force push и согласования с командой.

### Проверка файлов в LFS

```bash
# Просмотр файлов, хранящихся в LFS
git lfs ls-files

# Проверка, находится ли файл в LFS
git lfs pointer --file=path/to/file
```

## Клонирование репозитория с LFS

При клонировании репозитория с LFS файлами:

```bash
# Обычное клонирование (загружает LFS файлы)
git clone <url>

# Клонирование без загрузки LFS файлов (быстрее)
GIT_LFS_SKIP_SMUDGE=1 git clone <url>

# Загрузить LFS файлы после клонирования
git lfs pull
```

### Получение конкретных LFS файлов

```bash
# Получить все LFS файлы
git lfs fetch --all
git lfs checkout

# Получить LFS файлы для конкретной ветки
git lfs fetch origin <ветка>
git lfs checkout
```

## Управление LFS файлами

### Просмотр информации

```bash
# Статус LFS
git lfs status

# Информация о LFS
git lfs env

# Размер LFS файлов
git lfs ls-files | awk '{sum+=$3} END {print sum/1024/1024 " MB"}'
```

### Очистка

```bash
# Удалить локальные LFS файлы (оставить указатели)
git lfs prune

# Удалить старые версии LFS файлов
git lfs prune --verify-remote
```

## Настройка .gitattributes для проектов 1С

Рекомендуемая конфигурация для проектов 1С:

```gitattributes
# Выгрузки конфигурации
*.erf filter=lfs diff=lfs merge=lfs -text
*.cf filter=lfs diff=lfs merge=lfs -text

# Дампы баз данных
*.dt filter=lfs diff=lfs merge=lfs -text

# Архивы
*.zip filter=lfs diff=lfs merge=lfs -text
*.7z filter=lfs diff=lfs merge=lfs -text
*.rar filter=lfs diff=lfs merge=lfs -text

# Большие бинарные файлы
*.bin filter=lfs diff=lfs merge=lfs -text

# Медиафайлы (если большие)
*.mp4 filter=lfs diff=lfs merge=lfs -text
*.avi filter=lfs diff=lfs merge=lfs -text

# Исключения для небольших файлов
!*.bsl filter=lfs diff=lfs merge=lfs -text
!*.xml filter=lfs diff=lfs merge=lfs -text
```

## Ограничения и квоты

### GitHub

- Бесплатные аккаунты: 1 GB хранилища, 1 GB трафика в месяц
- Платные планы: большие лимиты

### GitLab

- Бесплатные проекты: 10 GB хранилища
- Платные планы: большие лимиты

### Собственный сервер

При использовании собственного Git сервера ограничения определяются доступным дисковым пространством.

## Рекомендации для 1С программистов

### Что хранить в LFS

✅ **Рекомендуется:**

- Выгрузки конфигурации (.erf) большого размера
- Дампы баз данных для тестирования
- Большие архивы с внешними обработками
- Медиафайлы, используемые в проекте

❌ **Не рекомендуется:**

- Исходный код (.bsl, .xml файлы)
- Небольшие файлы конфигурации
- Файлы, которые часто изменяются

### Альтернативы для больших файлов

Рассмотрите альтернативы Git LFS:

1. **Артефакты CI/CD** — хранить большие файлы как артефакты сборки
2. **Облачное хранилище** — использовать S3, Azure Blob и т.д.
3. **Отдельные репозитории** — вынести большие файлы в отдельный репозиторий
4. **Ссылки вместо файлов** — хранить только ссылки на файлы в документации

### Рабочий процесс

```bash
# 1. Настроить отслеживание
git lfs track "*.erf"

# 2. Добавить .gitattributes
git add .gitattributes

# 3. Добавить файлы
git add config.erf

# 4. Коммит
git commit -m "feat: Добавлена выгрузка конфигурации"

# 5. Push (LFS файлы загрузятся автоматически)
git push
```

## Устранение проблем

### Проблема: LFS файлы не загружаются

**Решение:**

```bash
# Проверить установку LFS
git lfs version

# Переустановить hooks
git lfs install --force

# Загрузить файлы вручную
git lfs pull
```

### Проблема: Превышен лимит хранилища

**Решение:**

1. Проверить размер LFS файлов: `git lfs ls-files`
2. Удалить старые версии: `git lfs prune`
3. Рассмотреть альтернативные способы хранения

### Проблема: Медленная работа

**Решение:**

1. Использовать shallow clone для истории
2. Загружать LFS файлы выборочно
3. Использовать локальный кэш LFS
