## Общие сведения

Файл `.gitignore` определяет, какие файлы и папки Git должен игнорировать и не отслеживать. Правильная настройка `.gitignore` критически важна для проектов 1С, чтобы исключить временные файлы, служебные папки и другие файлы, которые не должны попадать в репозиторий.

## Создание .gitignore

Создайте файл `.gitignore` в корне репозитория:

```bash
touch .gitignore
```

Или создайте файл через редактор в корне проекта.

## Специфичные паттерны для 1С

### Исключение конкретных файлов

```gitignore
# Игнорировать конкретный файл
config.ini

# Игнорировать файлы в конкретной папке
temp/config.ini

# Игнорировать все файлы определенного типа в папке
src/*.tmp
```

### Исключение папок

```gitignore
# Игнорировать всю папку
build/
dist/
out/

# Игнорировать папку, но отслеживать .gitkeep
build/*
!build/.gitkeep
```

### Условное игнорирование

```gitignore
# Игнорировать файлы, но не папки с таким именем
*.log
!important.log

# Игнорировать все в папке, кроме определенных файлов
temp/*
!temp/.gitkeep
!temp/readme.txt
```

## Проверка .gitignore

### Проверка игнорируемых файлов

```bash
# Показать все игнорируемые файлы
git status --ignored

# Проверить, будет ли файл игнорироваться
git check-ignore -v path/to/file
```

### Принудительное добавление игнорируемого файла

Если нужно добавить файл, который находится в `.gitignore`:

```bash
git add -f path/to/file
```

## Удаление уже отслеживаемых файлов

Если файлы уже были добавлены в Git до настройки `.gitignore`:

```bash
# Удалить файлы из индекса (но оставить в файловой системе)
git rm --cached -r folder/

# Или для конкретного файла
git rm --cached file.log

# Затем закоммитить
git commit -m "chore: Удалены временные файлы из отслеживания"
```

## Глобальный .gitignore

Для настройки общих правил для всех репозиториев:

1. Создайте файл `.gitignore_global` в домашней директории:

```bash
# Windows
C:\Users\YourName\.gitignore_global

# Linux/macOS
~/.gitignore_global
```

2. Добавьте в него общие правила:

```gitignore
# Системные файлы
.DS_Store
Thumbs.db
desktop.ini

# Временные файлы
*.tmp
*.bak
*.swp
*~
```

3. Настройте Git для использования этого файла:

```bash
git config --global core.excludesfile ~/.gitignore_global
```

## Рекомендации для 1С программистов

### Что обязательно исключать

✅ **Всегда исключайте:**

- Файлы баз данных (`.1CD`)
- Временные файлы и логи
- Служебные папки платформы 1С
- Персональные настройки IDE
- Системные файлы

### Что можно включить

✅ **Можно включить:**

- Исходный код (`.bsl`, `.xml`)
- Файлы конфигурации проекта (если общие)
- Документацию
- Скрипты сборки

### Что требует решения

❓ **Требует решения:**

- Выгрузки конфигурации (`.erf`) — использовать Git LFS или отдельное хранилище
- Внешние обработки (`.epf`) — хранить исходники или бинарники
- Дампы баз данных — обычно не хранить в Git

### Примеры для разных сценариев

#### Разработка расширения

```gitignore
# Исходники расширения отслеживаются
# Но исключаем служебные файлы
*.log
*.tmp
1cv8/
.vscode/settings.json  # Персональные настройки
```

#### Управление версиями конфигурации

```gitignore
# Исходники в Git
# Выгрузки через LFS или отдельно
*.1CD
*.log
1cv8/
# *.erf  # Если используете LFS, уберите эту строку
```

#### Разработка внешних обработок

```gitignore
# Храним исходники, исключаем скомпилированные
*.epf
*.log
*.tmp
1cv8/
```

## Частые ошибки

### Ошибка: Файлы все еще отслеживаются

**Проблема:** Добавили правило в `.gitignore`, но файлы все еще отслеживаются

**Решение:**

```bash
git rm --cached -r .
git add .
git commit -m "chore: Применен .gitignore"
```

### Ошибка: Игнорируется нужный файл

**Проблема:** Нужный файл попадает под правило `.gitignore`

**Решение:**

```bash
# Добавить исключение перед общим правилом
!important.log
*.log
```

Или принудительно:

```bash
git add -f important.log
```
